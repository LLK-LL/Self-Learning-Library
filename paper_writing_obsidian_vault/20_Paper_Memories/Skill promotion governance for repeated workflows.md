# Skill promotion governance for repeated workflows

## Memory Content

The current memory mechanism separates RAG memory from skills: memory records what
happened, while skills define how to handle recurring task classes. This project
should adopt that mechanism for the paper knowledge base, but only for
workflow-governance and executable processing flows.

Paper-specific facts, mentor feedback, evidence, manuscript claims, and writing
rules should remain in the local paper knowledge base. Repeated operational
flows may become skills only after they have stable triggers, ordered steps,
validation checks, and failure branches.

## Reusable Rule

Treat repeated workflow knowledge as a skill candidate, not as a manuscript rule.
Use lifecycle stages P0 memory, P1 promotable, P2 draft skill, P3 active skill,
and P4 revise or retire.

## Boundary

This is knowledge-base governance. It must not become a manuscript-writing
requirement.
