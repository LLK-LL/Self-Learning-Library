# Memory

The user wants future manuscript-formatting tasks to record not only the final formatting result, but also the formatting errors discovered during inspection and the concrete solution used to fix them.

# Application

For future formatting work:

- record the visible error, such as non-centred equations, wrong subscripts, missing characters, damaged placeholders, tense violations, spacing drift or heading-format mismatch
- record the technical cause when it can be identified, such as damaged OMML, stray inline math objects, inherited Word styles or direct paragraph overrides
- record the applied fix, such as rebuilding OMML, removing inline formula remnants, resetting paragraph properties or applying the reference article style
- write the reusable lesson into the paper knowledge base before closing the task
