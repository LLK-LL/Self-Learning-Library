# RAG retrieval should be a selective loading layer

## Rule Type

- `process_governance`

## Governance Rule

RAG retrieval should shortlist relevant knowledge-base notes and reduce context loading. It must not replace full evidence review for final generalization, conflict resolution, or manuscript rule promotion.

## Boundary

- This rule governs the knowledge-base workflow only.
- It must not be treated as a manuscript claim, section-structure requirement, wording rule, or paper evaluation criterion.

## Evidence Memories

- [[Before creating scripts retrieve and reuse local scripts]]
- [[KB RAG should retrieve selectively and preserve evidence boundaries]]
- [[Writing-time retrieval should stop at paper memories]]

## Governance Test

For ordinary writing, use RAG to choose candidate notes first; for final summarization or conflict handling, verify the selected notes against full vault evidence before promotion.
