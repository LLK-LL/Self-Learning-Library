# Contributing to Self-Learning Library

Self-Learning Library is a local-first harness for reusable AI working knowledge. Contributions should help agents retrieve relevant rules, avoid regressions, and promote repeated lessons into stable knowledge.

## Good Contributions

- Domain vault templates for coding, research, product, lab workflows, legal drafting, or personal SOPs.
- Improvements to lightweight retrieval quality.
- Tests for rule promotion and conflict detection.
- Clear examples of no-regression guards.
- Safer publishing and privacy guidance.
- Cross-platform scripts for macOS and Linux.
- Documentation that makes the harness easier to adapt.

## Keep the Project Focused

Please keep contributions aligned with:

- local-first Markdown knowledge bases;
- inspectable rule and memory layers;
- evidence-grounded rule promotion;
- selective retrieval instead of whole-vault prompting;
- no-regression checks;
- simple scripts over heavy infrastructure.

Avoid turning the project into a hosted SaaS, a general vector database, or a large agent framework.

## Before Opening a Pull Request

1. Keep the change focused.
2. Do not include private manuscripts, raw chat logs, credentials, or personal data.
3. Use synthetic examples when possible.
4. Update README or examples if behavior changes.
5. Explain any changes to retrieval, iteration, or rule promotion.

## Pull Request Checklist

- [ ] The change is scoped and easy to review.
- [ ] No private data or secrets are included.
- [ ] Documentation links were checked.
- [ ] New workflow behavior is explained.
- [ ] Safety implications are mentioned when relevant.
